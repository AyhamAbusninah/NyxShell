# Security Policy

## Supported Versions

NyxShell is an active systems engineering project. We currently support the following versions with security updates and patch fixes:

| Version | Supported          | status             |
| ------- | ------------------ | ------------------ |
| 1.0.x   | :white_check_mark: | Active Development |
| < 1.0   | :x:                | Deprecated         |

## Reporting a Vulnerability

We take the security of NyxShell seriously. If you discover a vulnerability (e.g., buffer overflows, race conditions, or privilege escalation issues), please report it responsibly.

### How to Report

Please do **not** open a public GitHub issue for sensitive security vulnerabilities. Instead, report them privately via email to the maintainers:

* **Ayham Abusnineh**: [a079.ayham@gmail.com]
* **Jihad Aljubeh**: [aljubehjihad@gmail.com]

### What to Include

* Description of the vulnerability.
* Steps to reproduce the issue (POC code is appreciated).
* Any relevant trace logs or Valgrind output.

### Response Timeline

* We will acknowledge receipt of your report within 48 hours.
* We will provide an estimated timeline for a fix within 1 week.
* Once fixed, we will credit you in the release notes (unless you prefer anonymity).
